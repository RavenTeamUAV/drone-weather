# ✈ Drone Weather — Перевірка погоди для місій БПЛА

Веб-застосунок для перевірки метеоумов перед вильотом.
Завантажте файл місії з Mission Planner → оберіть борт → отримайте рішення **ДОЗВОЛЕНО / ОБЕРЕЖНО / НЕ ЛЕТІТИ**.

---

## Що потрібно встановити

| Що | Посилання | Навіщо |
|----|-----------|--------|
| **Docker Desktop** | https://www.docker.com/products/docker-desktop/ | Запускає застосунок в ізольованому середовищі |

> Node.js встановлювати не потрібно — Docker бере все з собою.

---

## Запуск (3 команди в терміналі)

```bash
# 1. Розпакуйте архів і перейдіть в папку
cd drone-weather

# 2. Запустіть через Docker
make docker

# 3. Відкрийте браузер
# http://localhost:3000
```

**Зупинити:**
```bash
make docker-stop
```

---

## Якщо Docker ще не встановлено

1. Перейдіть на https://www.docker.com/products/docker-desktop/
2. Завантажте версію для вашої ОС (Mac / Windows / Linux)
3. Встановіть і запустіть Docker Desktop
4. Поверніться до команд вище

---

## Якщо хочете запустити без Docker (потрібен Node.js ≥ 18)

```bash
make setup    # одноразово — встановлює залежності
make start    # запуск сервера
```

---

## Як користуватися

1. **Місія** — натисніть "Завантажити .waypoints" → виберіть файл з Mission Planner
2. **Борт** — оберіть зі списку або натисніть **＋** щоб додати свій
3. **Час вильоту** — вкажіть дату/час або натисніть "Зараз"
4. **Точка зльоту** — ставиться автоматично з місії, або клікніть на карті
5. **Перевірити погоду** — отримайте результат по кожній точці маршруту

### Результати
- 🟢 **ДОЗВОЛЕНО** — всі параметри в нормі
- 🟡 **ОБЕРЕЖНО** — є граничні значення, рішення за екіпажем
- 🔴 **НЕ ЛЕТІТИ** — перевищено ліміти борту або небезпечна погода

### Що перевіряється
- Вітер на зльоті (10м) та по маршруту (на висоті кожної точки)
- Пориви вітру
- Хмарність низька (0–3 км)
- Видимість
- Ізотерма 0°C (ризик обмерзання)
- Опади
- Температура відносно ТТХ борту

---

## Додавання власного борту

Натисніть **＋** біля вибору борту → заповніть форму → **Зберегти борт**.
Борт зберігається в `data/drones.json` і залишається після перезапуску.

---

## Команди Makefile

```
make docker       — зібрати і запустити (Docker)
make docker-stop  — зупинити
make logs         — переглянути логи
make setup        — встановити npm пакети (без Docker)
make start        — запустити локально (без Docker)
make stop         — зупинити локальний сервер
```
